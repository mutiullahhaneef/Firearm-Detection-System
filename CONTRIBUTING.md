# Contributing

## Reporting Issues

Search [existing issues](../../issues) before opening a new one. Include steps to reproduce, expected vs. actual behavior, and your Python version and OS.

## Submitting a Pull Request

1. Fork the repository and create a branch from `main`.
2. Make focused, well-scoped commits.
3. Run tests before submitting: `pytest tests/`
4. Open a pull request with a clear description of the change.

## Code Style

- Follow [PEP 8](https://pep8.org/).
- Keep lines under 100 characters.
- Add docstrings to public functions.

## Commit Messages

```
feat: add real-time video inference
fix: correct bounding box scaling on hi-DPI screens
docs: update quick-start guide
refactor: extract detection logic into service layer
```

## Ethical Use

All contributions must comply with the ethical use terms in the [LICENSE](LICENSE). Do not add features intended for unauthorized surveillance or harmful purposes.
